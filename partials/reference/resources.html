Resources

Resources are what actually provide Services. By default, every User is a Person Resource for their own Services. You can create additional types of Resources for Services via the API.

For more information on what a Resource is and how to use them see Entities Resources.

Get Resources

This will look up all Resources that your account has access to - by default this includes all Resources under your own User account as well as any Resources of Users for which you are the Account Manager. Provide user_id of a managed User to return only the Resources for that User.

{% highlight ruby %} GET /resources {% endhighlight %}
Parameter Type Description Required?
user_id Integer Provide a User ID to see only this Userâ€™s Resources
page Integer Result page of interest, default: 1
per_page Integer Number of results per page, default: 10

Get Resource by ID

Look up details for a Resource by ID. You can access details for any Resource under your own User account as well as any Resources of Users for which you are the Account Manager.

{% highlight ruby %} GET /resources/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of Resource to retrieve

Create a Resource

Create a new Resource. By default, your own User account will automatically become the Account Manager for any newly created Resources.

{% highlight ruby %} POST /resources/new {% endhighlight %}
Parameter Type Description Required?
name String Name of Resource being created
type String Must be a valid Resource type. See Entities Resources.
user_id Integer Provide a User ID of a managed User to create a Resource for this User
gender String Gender of Resource, if applicable
capacity Integer Capacity of Resource, if applicable
description String Description of Resource
thing_type_id Integer When Resource is a Thing, the type Id

Update a Resource

Update information for a Resource. The ID must be of a Resource that is under your User account or any User for which you are the Account Manager. You can not update Resources that were previously deleted.

{% highlight ruby %} PUT /resources/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of Resource to update
name String New name of Resource
gender String Gender of Resource, if applicable
capacity Integer Capacity of Resource, if applicable
description String Description of Resource
thing_type_id Integer When Resource is a Thing, the type Id

Delete a Resource

Delete a Resource by ID. The ID must be of a Resource that is under your User account or any User for which you are the Account Manager. Deletion is a multi-step process: first the Resource is marked inactive and no longer appears in Search results; if no Bookings exist for this Resource then it is completely destroyed, otherwise it is kept archived for historical purposes. You can not delete Resources that were previously deleted.

{% highlight ruby %} DELETE /resources/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of Resource to delete
