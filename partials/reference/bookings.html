Bookings

The main goal of GoneBusy is to connect users with each other via bookings. A Booking represents the commitment by one user to attend a Service/Activity of another user at a particular time. The API allows the creation and management of Bookings.

Get Bookings

Look up all uncompleted Bookings that your account has access to - this includes all Bookings under your own User account as well as any Bookings of Users for which you are the Account Manager.

{% highlight ruby %} GET /bookings {% endhighlight %}
Parameter Type Description Required?
user_id Integer Provide a User ID to see only this User’s Bookings
states String Provide a comma-separated list of Booking states in order to only retrieve Bookings in those states. Leave blank to retrieve all possible states.
page Integer Result page of interest, default: 1
per_page Integer Number of results per page, default: 10

Get Booking by ID

Look up details for a Booking by ID. You can access details for any Booking under your own User account as well as any Bookings of Users for which you are the Account Manager.

{% highlight ruby %} GET /bookings/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of booking to retrieve

Create a Booking

Create a new Booking. Your own User account will by default be recorded as the initiator of the booking. Optionally, provide a user_id for a User you are the Account Manager of in order to create a booking.

{% highlight ruby %} POST /bookings/new {% endhighlight %}
Parameter Type Description Required?
service_id Integer The ID of the Service that is to be booked
date String Provide the date of the requested Booking. Several formats are supported: "2014-10-31", "October 31, 2014"
time String Provide the time of the requested Booking. Several formats are supported: “9am”, “09:00”, “9:00”, “0900”
resource_id Integer The ID of a Resource to be booked. If not provided, the first available Resource will be booked.
duration Integer If the service allows requesting a variable amount of time, specify how many minutes are desired
user_id Integer ID of a User you are the Account Manager of for whom you would like to initiate a booking

Update a Booking

Update information for a Booking. The ID must be of a Booking that is under your User account or any User for which you are the Account Manager. You can not update appointment time information for a Booking once placed. This action is a placeholder for future update-able information.

{% highlight ruby %} PUT /bookings/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of booking to update

Cancel a Booking

Attempt to cancel a Booking by ID. The ID must be of a Booking that is under your User account or any User for which you are the Account Manager. If cancellation is not possible, this will return an error message.

{% highlight ruby %} DELETE /bookings/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of booking to cancel
