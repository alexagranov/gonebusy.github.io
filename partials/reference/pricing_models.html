Pricing Models

Each Service a User offers must have a Pricing Model associated with it in order to specify how to charge for the Service. There are several types of Pricing Models, including “Activity” which means the Service is actually an Activity that has no charge Associated with it.

For more information on what a Pricing Model is and how to use them see Entities Pricing Models.

Get Pricing Models

This will look up all PricingModels in the system.

{% highlight ruby %} GET /pricing_models {% endhighlight %}
Parameter Type Description Required?
user_id Integer Provide a User ID to see only this User’s Pricing Models
page Integer Result page of interest, default: 1
per_page Integer Number of results per page, default: 10

Get Pricing Model by ID

Look up details for a Pricing Model by ID. You can access details for any Pricing Model under your own User account as well as any Pricing Models of Users for which you are the Account Manager.

{% highlight ruby %} GET /pricing_models/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of pricing model to retrieve

Create a Pricing Model

Create a new Pricing Model. Your own User account will automatically become the Account Manager for any newly created Pricing Models.

{% highlight ruby %} POST /pricing_models/new {% endhighlight %}
Parameter Type Description Required?
name String Name of Pricing Model being created
type String Must be a valid Pricing Model type. See Entities Pricing Models.
user_id Integer Provide a User ID of a managed User to create a Pricing Model for this User
price Float Price, if applicable
currency String 3-letter ISO currency code
notes String Additional notes about Pricing Model

Update a Pricing Model

Update information for a Pricing Model. The ID must be of a Pricing Model that is under your User account or any User for which you are the Account Manager.

{% highlight ruby %} PUT /pricing_models/{id} {% endhighlight %}
Parameter Type Description Required?
id Integer ID of pricing_model to update
name String New name of Pricing Model
